How to Use

Essential Commands

Check Status

rails db:migration:status

View all migrations and their sync status

Rollback Orphaned

rails db:migration:rollback_orphaned

Interactively rollback orphaned migrations

Run Diagnostics

rails db:migration:doctor

Troubleshoot configuration issues

Workflows

Switching Branches

  1. 1. Check migration status before switching
  2. 2. Switch branch (auto-warning if issues)
  3. 3. Rollback orphaned if needed
  4. 4. Run missing migrations if any

Before Merging

  1. 1. Run status check
  2. 2. Ensure no orphaned migrations
  3. 3. Commit all migrations
  4. 4. Push with confidence

Configuration

Customize Migration Guard to fit your workflow with extensive configuration options: 

# config/initializers/migration_guard.rb
MigrationGuard.configure do |config|
  # Environments where MigrationGuard is active
  config.enabled_environments = [:development, :staging]
  
  # Git integration level
  config.git_integration_level = :warning  # :off, :warning, :auto_rollback
  
  # Branch configuration
  config.main_branch_names = %w[main master trunk]
  config.target_branches = %w[main develop staging]  # optional
  
  # Behavior options
  config.warn_on_switch = true
  config.block_deploy_with_orphans = false
  config.sandbox_mode = false
  
  # Cleanup policies
  config.auto_cleanup = false
  config.cleanup_after_days = 30
  
  # Logging
  config.log_level = :info
  config.colorize_output = true
end
